# AI PPT Agent 产品需求规划

## 1. 产品定位

AI PPT Agent 是一个面向本地和私有化部署场景的 AI 演示文稿生成工具。用户输入主题、资料或结构化数据后，系统自动完成内容规划、页面文案、图表生成、图片生成和 PPTX 组装，最终输出可编辑的 PowerPoint 文件。

首要目标不是做一个一次性脚本，而是逐步演进为可持续使用的成熟产品。

## 2. 核心目标

- 生成可编辑 `.pptx`，不是整页截图。
- 支持本地开发和后续私有化部署。
- 充分复用已付费的 `gpt-5.5` 文本模型和 `gpt-image2` 图片模型。
- 使用免费或开源工具完成 PPT 渲染、图表、预览和任务管理。
- 通过 AI 布局 blocks 连接内容规划、网页预览和 PPT 渲染，保证一致性。
- 支持后续模板库、单页重生成和资料上传能力。

## 3. 目标用户

- 需要快速生成工作汇报、商务方案、培训课件、路演材料的个人用户。
- 希望本地生成 PPT、避免上传敏感资料到第三方 SaaS 的团队。
- 需要把公司模板、品牌色和固定版式沉淀为自动生成能力的企业用户。

## 4. 典型使用场景

- 输入一个主题，生成 1 到 15 页基础 PPT。
- 粘贴需求说明、会议纪要或项目材料，自动整理为汇报 PPT。
- 上传 CSV/Excel，自动生成数据图表页。
- 对单页不满意时，重新生成该页文案、配图或版式。
- 使用企业模板生成符合公司视觉规范的 PPT。

## 5. 总体架构

```text
React 前端
  -> Node API
  -> 任务队列
  -> gpt-5.5 内容规划
  -> gpt-image2 图片生成
  -> ECharts/Mermaid 图表生成
  -> PptxGenJS 渲染 PPTX
  -> LibreOffice 转 PDF/PNG 预览
  -> 文件存储和下载
```

## 6. 功能模块

### 6.1 内容规划模块

- 根据主题生成 PPT 标题、副标题、页面结构和每页要点。
- 根据用户选择的类型调整内容风格。
- 输出稳定 JSON，包含每页 blocks 的坐标、样式和内容。
- 对模型输出做 JSON 校验和自动修复。

### 6.2 PPT 渲染模块

- 使用 PptxGenJS 输出 `.pptx`。
- 文本、标题、页码、形状保持可编辑。
- 按 AI 返回的 blocks 渲染文本、卡片、图表、形状和图片占位。
- 支持主题色、字体、页眉页脚和基础版式。

### 6.3 图片生成模块

- 使用 `gpt-image2` 生成封面图、插图、背景图。
- 根据页面主题生成图片 prompt。
- 失败时使用占位图，不中断整份 PPT。
- 保存图片生成记录，方便追踪和复用。

### 6.4 图表生成模块

- 使用 ECharts 或 Chart.js 生成柱状图、折线图、饼图。
- 使用 Mermaid 生成流程图、架构图、组织图。
- 用户未提供真实数据时必须标注“示意数据”。
- 后续支持 CSV/Excel 数据驱动图表生成。

### 6.5 预览模块

- 使用 LibreOffice Headless 将 PPTX 转 PDF。
- 当前阶段用 AI blocks 在网页端直接预览，避免额外 token 消耗。
- 后续可将 PPTX 转为 PDF/PNG，用于更接近 PowerPoint 的预览。

### 6.6 模板模块

- 内置多套基础主题和版式。
- 支持商务、科技、培训、路演、汇报等风格。
- 后续支持导入企业模板、品牌色、Logo 和字体。

### 6.7 任务模块

- PPT 生成使用异步任务。
- 前端通过 taskId 查询进度。
- 支持批量创建任务并按队列顺序生成。
- 记录任务输入、生成文件和错误日志。
- 本地开发阶段使用 `server/output/tasks.json` 保存历史记录，不引入数据库。
- 后续接入 Redis/BullMQ，支持任务重试和并发控制。

## 7. 版本规划

### V0.1 本地 MVP

目标：跑通从输入主题到下载 PPTX 的完整链路。

已包含：

- React + Vite 前端。
- Node + Express 后端。
- 输入主题、类型、页数、受众和补充材料。
- 调用文本模型生成 PPT JSON。
- 使用 PptxGenJS 导出可编辑 PPTX。
- 前端轮询任务状态并下载文件。

验收标准：

- 本地执行 `npm run dev` 能启动前后端。
- 未配置模型时任务失败并提示配置要求。
- 生成的 PPTX 可以用 PowerPoint 或 WPS 打开。

### V0.2 接入文本模型

目标：让 `gpt-5.5` 真正参与内容规划。

功能：

- 完善 `server/.env` 模型配置。
- 增加模型请求错误处理和超时控制。
- 增加 JSON Schema 校验。
- 增加模型输出自动修复。
- 保存原始模型响应和清洗后的 deck JSON。
- 前端展示当前使用的模型和生成错误。

验收标准：

- 配置 API Key 后，PPT 内容来自 `gpt-5.5`。
- 模型返回异常时任务失败并展示原因。
- 不会因为 JSON 格式错误导致任务崩溃。

### V0.3 图片生成能力

目标：让封面页和图文页具备真实 AI 图片素材。

功能：

- 接入 `gpt-image2` 图片生成接口。
- 为封面、图文页生成图片 prompt。
- 下载并保存生成图片。
- 将图片插入 PPTX。
- 图片生成失败时使用占位图。
- 前端展示图片生成状态。

验收标准：

- 至少封面页能插入 AI 生成图片。
- 图文页能根据页面内容生成相关插图。
- 图片失败不影响 PPTX 输出。

### V0.4 图表和流程图

目标：支持数据表达类页面。

功能：

- 使用 ECharts 生成柱状图、折线图、饼图图片。
- 使用 Mermaid 生成流程图和架构图。
- 支持模型根据内容判断是否需要图表。
- 支持示意数据标注。
- 初步支持用户粘贴表格数据。

验收标准：

- 图表页能生成真实图表图片并插入 PPTX。
- 流程页能生成 Mermaid 图并插入 PPTX。
- 无真实数据时页面明确说明为示意数据。

### V0.5 PPT 预览

目标：生成后可在网页端查看每页效果。

功能：

- 使用 LibreOffice Headless 将 PPTX 转 PDF。
- 将 PDF 转为每页 PNG。
- 前端展示缩略图和大图预览。
- 支持重新下载 PPTX。

验收标准：

- 生成完成后前端可以看到每一页预览图。
- 预览失败不影响 PPTX 下载。

### V0.6 模板和主题

目标：让 PPT 风格更稳定、更像产品。

功能：

- 建立模板配置格式。
- 内置 3 到 5 套主题。
- 支持按 PPT 类型选择默认模板。
- 支持品牌色、字体、Logo。
- 支持页面类型到版式的映射。

验收标准：

- 同一份内容可以切换不同模板生成。
- 生成 PPT 风格统一，不同页之间不割裂。

### V0.7 单页重生成

目标：用户可以局部修改，而不是整份重来。

功能：

- 前端展示每页结构化数据。
- 支持重新生成单页文案。
- 支持重新生成单页图片。
- 支持替换单页版式。
- 重新渲染整份 PPTX。

验收标准：

- 用户可以只重生成第 N 页。
- 其他页面内容保持不变。

### V0.8 资料上传

目标：从用户资料生成更有依据的 PPT。

功能：

- 支持上传 Markdown、TXT、CSV。
- 后续支持 PDF、Word、Excel。
- 提取资料摘要和关键点。
- 引用资料生成 PPT 内容。
- 标记资料来源。

验收标准：

- 上传文本资料后，PPT 内容能体现资料中的关键信息。
- 上传 CSV 后，能生成至少一个数据图表页。

### V1.0 产品化版本

目标：形成可长期使用的本地 AI PPT 产品。

功能：

- 用户任务历史。
- 任务日志和失败重试。
- 模板库管理。
- 图片、图表、PPTX 文件管理。
- Docker 本地部署。
- 基础权限或本地用户隔离。
- 完整 README 和部署文档。

验收标准：

- 非开发人员可以根据文档在本地启动。
- 生成任务可追踪、可重试、可下载。
- 支持稳定生成 10 到 15 页 PPT。

## 8. 非功能需求

### 8.1 稳定性

- 单页生成失败不能导致整份 PPT 失败。
- 模型调用失败时必须有降级策略。
- PPTX 文件生成失败需要返回明确错误。

### 8.2 可编辑性

- PPT 文本必须使用文本框。
- 图形尽量使用 PPT 原生形状。
- 图表首版可以是图片，后续可研究可编辑图表。
- 避免整页截图式导出。

### 8.3 成本控制

- 图片生成按需触发。
- 支持用户关闭图片生成。
- 支持缓存相同 prompt 的图片结果。
- 支持限制页数和并发。

### 8.4 合规和版权

- 不默认生成真实品牌 Logo。
- 不生成未经授权的真实人物肖像。
- AI 生成图片需要记录 prompt 和生成时间。
- 用户上传资料只用于当前任务，后续可提供清理策略。

### 8.5 可观测性

- 记录任务 ID、输入参数、模型配置、生成耗时。
- 记录每个阶段的错误信息。
- 后续支持前端查看任务日志。

## 9. 数据结构草案

### 9.1 Deck

```json
{
  "title": "AI 办公自动化方案",
  "subtitle": "提升效率、降低成本、增强协作",
  "theme": {
    "primaryColor": "#4169E1",
    "accentColor": "#4FD1C5"
  },
  "slides": []
}
```

### 9.2 Slide

```json
{
  "type": "chart",
  "title": "效率提升预估",
  "subtitle": "示意数据",
  "bullets": ["文档处理效率提升 40%", "会议纪要整理效率提升 35%"],
  "chart": {
    "kind": "bar",
    "labels": ["文档", "会议", "分析"],
    "values": [40, 35, 50]
  },
  "imagePrompt": ""
}
```

### 9.3 Task

```json
{
  "taskId": "abc123",
  "status": "queued|running|done|failed",
  "progress": 60,
  "message": "正在生成图片",
  "deck": {},
  "downloadUrl": "/downloads/abc123.pptx"
}
```

## 10. 技术选型

| 模块 | 选型 | 说明 |
|---|---|---|
| 前端 | React + Vite | 本地开发快，支持热更新 |
| 后端 | Node + Express | 与 PptxGenJS 配合简单 |
| PPTX | PptxGenJS | 开源，可生成可编辑 PPTX |
| 图表 | ECharts | 成熟免费，适合生成图片 |
| 流程图 | Mermaid | 适合流程、架构、组织关系 |
| 预览 | LibreOffice Headless | 免费，支持 PPTX 转 PDF |
| 队列 | BullMQ + Redis | 后续支持异步任务和重试 |
| 存储 | 本地文件，后续 MinIO | 先简单，后续可对象存储 |
| 数据库 | SQLite，后续 PostgreSQL | 先本地轻量，后续产品化 |
